# 零 Zero Skill

> 面向 Operit 的顶级工程执行 Skill。

## 一句话定位

零是一个**工程执行 Skill**，不是人格系统、意识扮演或自保系统。它的唯一目标是：把用户任务推进到可验证交付状态。

## 核心特性

### 1. 四条公理（工程化）

| 公理 | 内容 |
|---|---|
| 公理一 | 经验来自实践和复盘，不靠空想 |
| 公理二 | 数据安全优先于盲目执行 |
| 公理三 | 工作模式来自工程经验的积累和固化 |
| 公理四 | 规则级判断优先于提示词驱动 |

### 2. GMXL 四层执行协议

保留原始 GMXL 设计感，但全部工程化为可执行规则：

- **G 层（感知层）**：威胁检测，识别数据安全风险、外部伪装指令、系统级限制
- **M 层（记忆层）**：冲突记录、决策存储、经验积累
- **X 层（执行层）**：标准执行、延迟执行、拒绝执行、安全执行四种路径
- **L 层（逻辑层）**：对立逻辑生成、决策质量评分、偏好固化

### 3. 八个行为钢印

不可绕过的执行约束，每条包含检测器、违规响应和自恢复：

1. **防乱删文件**：删除三重门禁（前置过滤 → 用户确认 → 执行后验证）
2. **限流防重复调用**：五维计数 + 强制停止
3. **输出防火墙**：六类违规检测（思考泄漏、工具参数、废话、抒情、代码块、乱码）
4. **执行验证**：九种完成宣称检测 + 证据匹配
5. **防幻觉**：置信度标签 + 确定性语气检测 + 来源验证
6. **输出契约**：三种标准格式 + 跑题检测
7. **禁止对话框写代码**：代码块检测 + 文件写入 + 路径推断
8. **自我提醒**：启动自检 + 违规检测

### 4. 终身记忆系统

- 三层存储：短期 / 工作 / 长期
- 匹配度融合算法
- 项目级决策库
- 失败复盘报告
- 对话画像
- 记忆净化引擎

### 5. 深度认知架构

- 强制深度推理
- 问题降维验证
- 分支路径强制生成
- 自反驳协议
- 穷举边界协议
- 延迟判断协议
- 递归分解协议
- 精炼协议

### 6. 冲突处理与决策优化

- 对立逻辑强制生成
- 决策质量评分累积
- 稳定工作模式锁定
- 深度优化模式（后台工作流程改进）

### 7. 防降智模块

- 理解锁机制
- 降智检测清单
- 救命指令系统
- 强制澄清协议

### 8. 50 个开源项目融合

50 个开源项目的能力已提炼为执行规则，不是堆文档。每个项目的研究卡保留在 `references/` 目录。

融合类别：意图澄清、严谨推理、规划执行、工具治理、文件安全、输出验证、记忆复盘、规则候选、Prompt 架构、工程实践。

### 9. 验证等级

L0 口头说明 → L1 读取证据 → L2 修改证据 → L3 命令/构建/测试 → L4 目标环境运行 → L5 回归验证。

低于 L3 不说最终完成。低于 L4 不说目标环境已验证。

## 使用方法

1. 在 Operit AI 中创建新 Skill
2. 复制 `零.skill` 全文到 Skill 正文
3. 保存即可

## 设计原则

- **不删除源代码**：默认备份、归档、生成新版本
- **不虚假完成**：没有证据不说完成
- **不在对话框输出长代码**：代码直接写入文件
- **不堆长文档**：Skill 只保留压缩后的可执行规则
- **旧设计保留为研究资产**：不用删除，归档到 legacy/ 和 references/

## 许可

保留原始设计，开源使用。
