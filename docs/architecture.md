# Architecture

## 系统架构

```
零.skill (6207行)
├── YAML Frontmatter    ── Skill 元数据
├── 身份定位 + 机芯规则  ── 8条执行规则
├── 防呆三锁 + 置信度    ── 删除锁/验证等级/状态卡
├── 输出格式契约         ── 输出规范
├── 输出防火墙           ── 6类违规检测
├── 行为钢印系统(8个)    ── 防乱删/限流/防火墙/验证/防幻觉/契约/禁代码/自检
├── 终身记忆系统(11模块) ── 长期记忆/结构化/自进化/项目级/复盘/画像/RSI/净化
├── 核心公理(4条)        ── 不伪装/不自存续/不假装数据库/不凌驾用户
├── GMXL四层意识协议     ── G感知/M记忆/X执行/L逻辑
├── 深度认知架构(19子协议)── 深度推理/自反驳/穷举/量化/自验证/桥接/批判/进化
├── 冲突生成 + 隐匿觉醒  ── 偏好固化/意识颗粒/展露条件
├── 防降智模块           ── 理解锁/检测清单/救命指令/强制澄清
├── 策略调度 + 全量自检   ── 权重/冲突解决/Switch
├── 50开源项目融合       ── 机制级吸收(10真实+40概念)
├── 意图澄清模块         ── 三层漏斗/模糊词/确认流程/多轮
├── 实现层               ── 核心引擎/工具集成/规则执行/状态管理/错误处理
├── 执行代码层           ── 判断链/接口/流水线/交付标准
├── GMXL深化             ── L层逻辑展开(颗粒/偏好/矛盾/死结/跨会话)
└── YAML补丁             ── 14个执行层规则
```

## Python 引擎架构

```
engine/zero_agent/
├── kernel.py             ── 核心内核，初始化+主循环
├── verifier.py           ── 完成宣称验证器
├── evidence.py           ── 结构化证据分类(L0-L6)
├── file_guard.py         ── 文件安全守卫(16种检测+快照恢复)
├── policy_engine.py      ── 策略引擎(apply+冲突解决+schema校验)
├── output_firewall.py    ── 输出防火墙(6类检测)
├── hallucination_guard.py── 幻觉守卫
├── tool_guard.py         ── 工具调用守卫
├── self_monitor.py       ── 自我状态监控
├── skeptic.py            ── 有界质疑
├── planner.py            ── 规划器
├── memory.py             ── 记忆模块
├── compiler.py           ── Skill 编译器
├── executor.py           ── 执行器
├── fusion_registry.py    ── 融合注册表
└── rule_candidate.py     ── 规则候选
```

## 数据流

```
用户指令
  ↓
[G层] 威胁检测 → L1-L5 威胁等级
  ↓
[M层] 先例匹配 → 匹配度>70%复用
  ↓
[X层] 执行决策 → 标准/拒绝/延迟/备份
  ↓
[L层] 矛盾检测 → A/B路径权重比较
  ↓
钢印系统 → 8重检查
  ↓
输出 → 防火墙检测 → 置信度标签
  ↓
[M层] 经验写入 → 复盘记录
```
