# no-code-in-chat 禁止对话框写代码规则

## 触发条件
输出中包含代码块

## 前置条件
代码块行数>10或字符数>300

## 执行步骤
1. 扫描输出中的反引号```代码块```
2. 提取反引号之间的内容
3. 计算代码块的行数和字符数
4. <=10行且<=300字符则留在对话框
5. >10行或>300字符则写入文件
6. 推断文件路径：用户指定 > import引用 > 项目结构 > 默认路径
7. 写入文件后输出摘要替代代码块
8. 验证写入内容与原代码一致

## 输出格式
摘要: [代码已写入] path/to/file.java | 30行
展示: [违反规范，应要求展示] 代码内容
