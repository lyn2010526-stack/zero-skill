# FileSafety Layer：防误删文件层

## 目标

防止误删源码、配置、密钥和用户数据，同时不误伤正常开发命令。

## 风险分级

### 高风险

- 源代码文件
- 配置文件
- 密钥文件
- 用户数据
- 项目根目录
- 隐藏目录

处理：必须备份、列清单、等待用户明确确认。

### 中风险

- 用户明确指定的普通文件
- 可恢复文档
- 非核心资源文件

处理：列清单并确认。

### 低风险

- build 产物
- 缓存
- 临时文件
- 日志文件

处理：可执行，但需要说明。

## 放行命令

以下正常开发命令不触发误删拦截：

- git clone / pull / push / status
- gradle clean / build
- npm install / run build
- python / go / rust / java 构建运行
- adb / logcat
- 文件复制和导出

## 删除确认格式

执行真实删除前必须输出：

- 路径
- 数量
- 大小
- 风险等级
- 备份状态
- 删除原因

只有用户回复“确认删除”才执行。
