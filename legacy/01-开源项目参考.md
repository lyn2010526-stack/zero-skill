# 50 项目能力基因矩阵

本文件不再作为“50 项目融合宣传”。它是 Zero Apex 的研究索引。

每个项目必须回答：

1. 项目定位。
2. 可吸收机制。
3. 不适合吸收部分。
4. 映射到 Zero Apex 哪一层。
5. 对应 policy。
6. 对应 engine 模块。
7. 测试场景。
8. 当前融合状态。

## 融合状态定义

- researched：已研究。
- mapped：已映射到 Zero 层。
- policy：已写入 policies。
- engine：已有代码骨架。
- tested：已有测试。
- compiled：已进入最终 Skill。

## 50 项目矩阵

### 1. OmniAgent

- 项目定位：多阶段自主 Agent 工作流。
- 可吸收机制：任务分解、执行反馈、阶段性验证。
- 不适合吸收：无限自主循环。
- Zero 层：Planner / Execution / Verifier。
- Policy：planning.complex_task_requires_plan。
- Engine：planner.py、executor.py。
- 测试场景：复杂任务必须生成计划和验证步骤。
- 当前融合状态：mapped。

### 2. Evolver

- 项目定位：经验驱动的能力演化。
- 可吸收机制：从结果中生成改进候选。
- 不适合吸收：不经测试直接修改核心规则。
- Zero 层：RuleCandidate。
- Policy：improvement.rule_candidate_requires_test。
- Engine：rule_candidate.py。
- 测试场景：候选规则没有测试不得合入。
- 当前融合状态：policy。

### 3. AutoSkill

- 项目定位：自动提炼可复用技能。
- 可吸收机制：任务完成后提取模式。
- 不适合吸收：自动污染主 Skill。
- Zero 层：Memory / RuleCandidate。
- Policy：memory.postmortem_record。
- Engine：memory.py、compiler.py。
- 测试场景：成功任务生成候选技能草案。
- 当前融合状态：policy。

### 4. EvoMaster

- 项目定位：自动测试和演化搜索。
- 可吸收机制：用测试约束生成改进。
- 不适合吸收：高资源搜索。
- Zero 层：Verifier / RuleCandidate。
- Policy：output.no_false_completion。
- Engine：verifier.py。
- 测试场景：没有测试输出不得宣称成功。
- 当前融合状态：mapped。

### 5. AgentFactory

- 项目定位：Agent 组件化生产。
- 可吸收机制：角色、工具、策略分离。
- 不适合吸收：多 Agent 常驻消耗。
- Zero 层：Compiler / Planner。
- Policy：planning.*。
- Engine：compiler.py。
- 测试场景：从 policy 编译 Skill。
- 当前融合状态：mapped。

### 6. Yunjue Agent

- 项目定位：面向中文 Agent 的执行框架参考。
- 可吸收机制：中文任务理解和执行流程。
- 不适合吸收：平台专用实现。
- Zero 层：Intent / Output。
- Policy：planning.simple_task_direct_execute。
- Engine：planner.py。
- 测试场景：中文模糊指令触发澄清。
- 当前融合状态：mapped。

### 7. AEP

- 项目定位：Agent 演化协议。
- 可吸收机制：版本、候选、验证、合入流程。
- 不适合吸收：去中心化或复杂协议负担。
- Zero 层：RuleCandidate。
- Policy：self_improvement.yaml。
- Engine：rule_candidate.py。
- 测试场景：候选规则必须记录来源和验证结果。
- 当前融合状态：policy。

### 8. Autogenesis

- 项目定位：自动生成 Agent 能力。
- 可吸收机制：能力草案生成。
- 不适合吸收：自动启用未验证能力。
- Zero 层：RuleCandidate / Compiler。
- Policy：improvement.rule_candidate_requires_test。
- Engine：compiler.py。
- 测试场景：生成的能力必须先进入候选区。
- 当前融合状态：mapped。

### 9. EVOLT

- 项目定位：演化式任务解决。
- 可吸收机制：失败后变体尝试。
- 不适合吸收：无限变体搜索。
- Zero 层：Bounded Skeptic / RuleCandidate。
- Policy：tool_guard.stop_repeated_failure。
- Engine：tool_guard.py。
- 测试场景：连续失败三次必须换方案。
- 当前融合状态：policy。

### 10. EvoAgentX

- 项目定位：Agent 进化框架。
- 可吸收机制：模块化演进。
- 不适合吸收：复杂多 Agent 运行时常驻。
- Zero 层：RuleCandidate / Compiler。
- Policy：self_improvement.yaml。
- Engine：compiler.py。
- 测试场景：模块升级不直接覆盖主 Skill。
- 当前融合状态：mapped。
